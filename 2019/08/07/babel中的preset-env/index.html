什么是@babel/preset-env

@babel/preset-env是一个智能预设。它允许开发者使用最新的 JavaScript，不再需要手动管理plugins和 polyfill

注意：@babel/preset-env中不包含实验plugins

安装

配置

targets

目标环境

.browserslistrc

推荐使用.browserslistrc文件来配置目标环境

注意：当@babel/preset-env的targets属性或ignoreBrowserslistConfig属性未配置时，才会读取.browserslistrc文件

示例：

learn_babel/babel.config.js

learn_babel/.browserslistrc

package.json

也可以通过package.json来配置目标环境

示例：

learn_babel/babel.config.js

learn_babel/package.json

useBuiltIns

配置@babel/preset-env如何处理 polyfill

运行：

entry

entry表示开发者需要手动引入所有 polyfill，babel会自动剔除目标环境中已经原生支持的那部分 polyfill，剩余的 polyfill 将把目标环境和 ECMAScript 最新版之间的缝隙填充完整，不管源代码中是否使用到了对应的 polyfill

示例：

learn_babel/babel.config.js

源文件：

learn_babel/index.js

输出：

learn_babel/index.compiled.js

示例：

learn_babel/babel.config.js

源文件：

learn_babel/index.js

输出：

learn_babel/index.compiled.js

usage

usage表示babel会自动引入源代码中使用到的且目标环境不支持的那部分 polyfill

示例：源代码中使用到的，且目标环境不支持的

learn_babel/babel.config.js

源文件：

learn_babel/index.js

输出：

learn_babel/index.compiled.js

示例：源代码中使用到的，但目标环境已支持的

learn_babel/babel.config.js

源文件：

learn_babel/index.js

输出：

learn_babel/index.compiled.js

false

false表示babel不会自动添加 polyfill，也不会剔除目标环境已经支持的那部分 polyfill

示例：

learn_babel/babel.config.js

源文件：

learn_babel/index.js

输出：

learn_babel/index.compiled.js

ignoreBrowserslistConfig

是否忽略.browserslistrc文件和package.json中的browserslist属性

默认值：false
